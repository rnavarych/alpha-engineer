# domain-iot

IoT domain specialization plugin for Claude Code providing expert guidance on end-to-end IoT system design, embedded development, and connected device ecosystems.

## Agents

- **iot-solution-architect** - End-to-end IoT architecture design across device, connectivity, edge, cloud, and application layers
- **iot-firmware-advisor** - Embedded systems programming, RTOS, OTA updates, peripheral drivers, and power management

## Skills

| Skill | Description |
|-------|-------------|
| **mqtt-patterns** | MQTT topic hierarchy, QoS levels, retained messages, LWT, broker selection, MQTT 5.0 features |
| **device-management** | Device provisioning, firmware OTA updates, device twin/shadow, command-and-control, fleet management |
| **edge-computing** | Edge gateway design, local vs cloud processing, edge ML inference, containers on edge, offline operation |
| **iot-security** | Device identity (X.509, TPM), secure boot, TLS/DTLS, firmware signing, network segmentation, attestation |
| **time-series-data** | InfluxDB/TimescaleDB/Prometheus modeling, downsampling, retention policies, compression, Grafana dashboards |
| **protocol-advisor** | MQTT vs CoAP vs AMQP vs HTTP, Zigbee, Z-Wave, BLE, LoRaWAN, NB-IoT/LTE-M, protocol bridging |
| **iot-cloud-integration** | AWS IoT Core, Azure IoT Hub, Google Cloud IoT alternatives, data pipelines, rules engines |
| **digital-twin** | Twin modeling, state synchronization, predictive maintenance simulation, what-if analysis, 3D visualization |
