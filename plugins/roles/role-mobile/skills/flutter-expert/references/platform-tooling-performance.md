# Platform Targets, Tooling, Interop, AI & Performance

## When to load
Load when targeting Flutter for Web (WASM), Desktop, or Flame game engine; integrating on-device AI (firebase_ai, google_generative_ai, MediaPipe); using platform channels or FFI; generating code with Mason/very_good_cli/flutter_gen; or optimizing Flutter app performance with Impeller.

## Flutter for Web (WASM — Production Default)

`flutter build web --wasm` is the production build command in 2025+. CanvasKit and HTML renderers are legacy.

```bash
# Production build (WASM — default recommended)
flutter build web --wasm

# Legacy CanvasKit (fallback for browsers without WASM GC support)
flutter build web --web-renderer canvaskit

# Development
flutter run -d chrome
```

- **WASM renderer**: compiles Dart to WebAssembly — near-native performance, no JS VM overhead
- Requires browsers with WasmGC support: Chrome 119+, Firefox 120+, Safari 17.4+
- For legacy browser support, provide CanvasKit fallback via `flutter_bootstrap.js` detection
- `dart:js_interop` + `package:web` for all JS/DOM interop (replaces `dart:html`)
- Service Workers for offline support: `flutter_service_worker.js` generated by build
- `kIsWeb` constant for platform-specific web code branches

```dart
// Modern JS interop — works with WASM
import 'dart:js_interop';
import 'package:web/web.dart';

final input = document.querySelector('#email') as HTMLInputElement;
input.value = 'user@example.com';

// Extension type for custom JS interop
extension type AnalyticsJS._(JSObject _) implements JSObject {
  external void track(String event, JSAny? properties);
}
```

## Flutter for Desktop (macOS / Windows / Linux)

- `flutter build macos`, `flutter build windows`, `flutter build linux`
- `MenuBar` widget for native macOS menu bar
- `window_manager` package for window size, position, title bar customization
- Platform filesystem access via `path_provider` and `dart:io`
- Drag-and-drop support via `super_drag_and_drop`
- `multi_window` / `desktop_multi_window` for multiple window support
- Keyboard shortcuts: `Shortcuts` + `Actions` widgets with `LogicalKeySet`

## On-Device & Generative AI

### firebase_ai (Firebase AI Logic)
- `firebase_ai` package — production-ready Flutter SDK for Vertex AI in Firebase
- Runs inference via Firebase backend — no local model required
- Supports Gemini models (text, multimodal), streaming responses
- Automatically handles auth, quota, and model routing

```dart
import 'package:firebase_ai/firebase_ai.dart';

final model = FirebaseAI.googleAI().generativeModel(model: 'gemini-2.0-flash');

// Text generation
final response = await model.generateContent([
  Content.text('Summarize this order: ${order.description}'),
]);
print(response.text);

// Streaming
final stream = model.generateContentStream([Content.text(prompt)]);
await for (final chunk in stream) {
  print(chunk.text);
}

// Multimodal — image + text
final imageBytes = await File(imagePath).readAsBytes();
final response = await model.generateContent([
  Content.multi([
    DataPart('image/jpeg', imageBytes),
    TextPart('What product is shown in this image?'),
  ]),
]);
```

### google_generative_ai (Direct Gemini API)
- `google_generative_ai` package for direct Gemini API access without Firebase
- Use for apps where Firebase is not in the stack
- `GenerativeModel`, `ChatSession` for multi-turn conversations
- Function calling support for tool integration

```dart
import 'package:google_generative_ai/google_generative_ai.dart';

final model = GenerativeModel(
  model: 'gemini-2.0-flash',
  apiKey: const String.fromEnvironment('GEMINI_API_KEY'),
  generationConfig: GenerationConfig(temperature: 0.7),
);

final chat = model.startChat(history: [
  Content.text('You are a helpful order assistant.'),
]);

final response = await chat.sendMessage(Content.text('Track order #123'));
```

### MediaPipe Tasks for Flutter (On-Device Inference)
- Google's cross-platform ML framework — runs entirely on-device, no network required
- Flutter: via `google_mediapipe` Flutter packages (vision, text, audio tasks)
- Zero latency, privacy-preserving (data never leaves device)

```dart
import 'package:mediapipe_task_vision/mediapipe_task_vision.dart';

final classifier = await ImageClassifier.create(
  options: ImageClassifierOptions.fromAsset(
    'assets/models/efficientdet_lite0.tflite',
    maxResults: 5,
    scoreThreshold: 0.5,
  ),
);

final result = classifier.classify(mpImage);
for (final classification in result.classifications.first.categories) {
  print('${classification.categoryName}: ${classification.score}');
}
classifier.close();
```

**Available Tasks:** Image classification, object detection, hand landmark detection,
pose estimation, face mesh, text classification, language detection, audio classification.

### On-Device AI Patterns
- Lazy model load: init model on first use, not at app startup
- Background model download with progress UI using `Dio` + `StreamController`
- Model versioning: server-driven refresh without app update
- Fallback: on-device → firebase_ai → graceful error message
- Cache inference results for identical inputs with `lru_cache` or custom map

## Flame (Game Engine)

- `FlameGame` as the root game class with `update(dt)` and `render(canvas)` lifecycle
- `Component` system: `PositionComponent`, `SpriteComponent`, `TextComponent`, `ParticleComponent`
- Collision detection: `HasCollisionDetection` mixin with `ShapeHitbox`
- Input: `TapCallbacks`, `DragCallbacks`, `KeyboardHandler` mixins on components
- Audio: `FlameAudio.play('sound.mp3')` with cached pre-loading
- Tiled map support via `flame_tiled` for 2D tile-based maps
- Physics via `flame_forge2d` (Box2D wrapper)
- Flame 1.x (stable): compatible with Flutter 3.x, Impeller-accelerated rendering

## Backend with Dart

### Serverpod 2.x
- Full-stack Dart framework: server, client SDK, and database ORM from one schema
- `protocol.yaml` defines endpoints, models, and database tables
- Generated client SDK for direct typed calls: `client.endpointName.methodName(args)`
- Serverpod 2: built-in authentication, file upload, real-time via WebSockets
- Horizontally scalable with Redis for session storage and pub/sub

### Dart Frog
- File-based routing: `routes/users/[id].dart` → `GET /users/:id`
- Middleware via `_middleware.dart` files for auth, logging, CORS
- `dart_frog build` for Docker image generation
- Edge deployment via Vercel Edge Functions or Cloudflare Workers

## Code Generation & CLI Tools

### Mason
- Template-based code generation with "bricks"
- `mason make feature_brick --name login` to scaffold from template
- `brick.yaml` defines variables; `__brick__/` directory contains templates
- `mason bundle` to publish bricks to brickhub.dev

### very_good_cli
- `very_good create flutter_app my_app` generates opinionated project structure
- Includes: Bloc, l10n, testing setup, CI/CD workflows, lint rules
- `very_good test --coverage --min-coverage 100` for coverage-gated CI

### flutter_gen (Asset Code Generation)
- Generates type-safe Dart classes for assets (images, fonts, colors, SVGs)
- Eliminates magic string paths like `'assets/images/logo.png'`
- `Assets.images.logo.image()` with compile-time safety

```yaml
# pubspec.yaml
flutter_gen:
  output: lib/gen/
  integrations:
    flutter_svg: true

# Generated usage
import 'package:my_app/gen/assets.gen.dart';
Image.asset(Assets.images.logo.path)
Assets.images.logo.image(width: 100)
Assets.fonts.inter  // font family constant
```

### DCM — Dart Code Metrics
- **DCM is a paid tool** (dcm.dev) for teams requiring advanced metrics
- `dcm analyze` for metrics report; `dcm check-unused-code` for dead code
- Free alternative: `dart_code_linter` (open-source fork of DCM)
- Rules: function length, cyclomatic complexity, number of parameters, coupling
- Built-in `dart analyze` covers most critical checks without extra tooling

### custom_lint
- Project-specific lint rules in Dart via a separate analyzer plugin
- `RiverpodLint` is the most common consumer — enforces Riverpod usage patterns
- **Performance warning**: `custom_lint` plugins run as a separate process and communicate with the Dart analyzer via sockets. On large projects (200k+ LOC) this causes measurable IDE lag and slower `dart analyze` times. Profile before committing.
- Dart SDK 3.x improved native analyzer plugin support — verify if `custom_lint` is still needed for your specific rules before adding it
- Works alongside `analysis_options.yaml`; add only when the lint value exceeds the performance cost

### Pub Workspaces (Dart 3.5+)
- Monorepo support: multiple packages share a single `pub get` resolution
- `pubspec.yaml` workspace declaration replaces `melos` for simple monorepos
- `melos` still preferred for large monorepos with complex script orchestration

## Platform Channels & Interop

### Pigeon (Recommended)
- Type-safe code generation for platform channel APIs
- `@HostApi()` for Dart-to-native calls; `@FlutterApi()` for native-to-Dart callbacks
- Generates Swift, Kotlin, Java, and Objective-C stubs from a Dart interface definition
- Run: `dart run pigeon --input pigeons/api.dart`

### FFI (Foreign Function Interface)
- `dart:ffi` for direct C library calls — zero overhead vs. method channels
- `ffigen` generates Dart bindings from C headers automatically
- `dart:ffi` NativeCallable for C calling Dart functions (callbacks)

### Method Channels (Legacy — prefer Pigeon)
- `MethodChannel('com.example/channel')` for async platform communication
- `EventChannel` for continuous streams (sensor data, BLE characteristic notifications)

## Flutter Hooks & freezed

### Flutter Hooks
- `HookWidget` base class with access to `useEffect`, `useState`, `useMemoized`, `useRef`
- `useAnimationController(duration:)` creates and disposes `AnimationController` automatically
- `useTextEditingController()`, `useFocusNode()`, `useScrollController()` for common controllers
- `useRef` for mutable values that don't trigger rebuilds

### freezed
- Immutable model classes with `copyWith`, pattern matching, `==`, `hashCode`
- `@freezed` annotation with `factory` constructors for union types
- `fromJson`/`toJson` via `json_serializable` integration
- Use native Dart 3 sealed classes for simple state unions; `freezed` for complex unions with JSON serialization

## Performance Checklist (Impeller Era)

### Impeller (Default on iOS and Android)
- Impeller is the **default renderer** since Flutter 3.22 on Android, 3.19 on iOS
- Pre-compiled shaders — zero first-frame jank (no Skia shader stutter)
- Metal on iOS, Vulkan on Android — consistent 60/120fps
- No need for SkSL shader pre-warming (was required with Skia)
- Verify: `flutter run --enable-impeller` (explicit); `--no-enable-impeller` to disable for debugging

### Widget Rebuild Optimization
- Profile with Flutter DevTools: Widget rebuild tracker, frame timeline, memory, network
- Avoid rebuilding entire widget trees — use `const`, `BlocSelector`, `Consumer` (Riverpod), `select`
- `RepaintBoundary` around frequently animated or complex subtrees to isolate repaints
- Minimize `Opacity` widget usage — prefer `FadeTransition` or `AnimatedOpacity`

### List Performance
- `ListView.builder` / `GridView.builder` for lazy list construction
- Set `itemExtent` for fixed-height items — enables O(1) scroll position calculation
- Cache network images with `cached_network_image` and `CachedNetworkImageProvider`

### Isolates
- `Isolate.run()` for CPU-intensive tasks (JSON parsing, image processing, crypto)
- `compute()` for simpler one-shot heavy computations
- Never perform heavy work on the main isolate

### Startup Performance
- `flutter_native_splash` for immediate branded splash (avoids white flash)
- Deferred imports: `import 'heavy_feature.dart' deferred as feature;`
- `flutter build apk --split-per-abi` to reduce download size

### Frame Budget (DevTools)
```
60fps target:  16ms/frame — Build <8ms | Layout <4ms | Paint <4ms
120fps target:  8ms/frame — common on ProMotion iOS / high-refresh Android
Raster thread jank: expensive custom painters, shadows, clip layers
Main isolate jank: JSON parsing, image decoding, crypto, regex
```
